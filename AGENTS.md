## Learned User Preferences
- Prefer maximum native timeline granularity for game replays; do not downsample frames just to shrink JSON when positioning fidelity matters. Prefer fast ROFL→JSONL ingest as a core workflow (offline/packet-driven when quality gates hold) over multi-tens-of-minutes Replay API capture; ROFL tooling should run on macOS and Windows.
- Prefer combat/team-win results shown visually (who wins / strength bands), not only raw damage numbers; treat pBlue/pRed as heuristic model scores (“model edge”), never as calibrated win probabilities or odds %.
- Map → calculator flow should support multi-champion NvM selection, including selecting both teams at once.
- Map review should support zoom/pan and show turrets, inhibitors, nexus, and jungle camps with official LoL wiki icons, clear alive/dead availability at second precision, and larger structure/camp icons than champion markers; fog of war should distinguish unseen-by-anyone vs unseen-by-opponents, and vision presence should inform skillshot xH.
- Dead champions must be excluded from calculator imports (do not count their combat stats); missing HP/combat/career fields from replay sources must stay unknown rather than faked as dead/full/zero; never fixture-remap another match’s HP/combat/ranks onto a real match; keep calculator import/Send disabled until HP, ability ranks, and combat stats are trustworthy; legitimate early liveclient score zeros are fine when the source marks them known.
- Action timelines must respect remaining HP via realistic survival/CD/cast/death timing — never absolute HP% bands that ban ability slots or cliff auto counts (low HP is not “one AA max”); a fighter who would die mid-window still cannot complete post-lethal actions.
- Prefer a competitive-style map scoreboard (towers, gold with delta, kills, role quests, grubs, dragons, baron, elder) and include objective buffs in combat math.
- Do not use Obsidian vault or recall for this workspace; project context lives in-repo (docs/, AGENTS.md, code).
- Prefer dark Anthropic/Apple-like product-tool UI that feels integrated, not a collage of disconnected assets; avoid em dashes in user-visible copy.
- Prefer extended-fight modeling with resource regeneration, calibrated against actual match outcomes from the timeline feed.
- Champion-history board should separate AS from AH, show gold as total/items/bag when known, quantify cumulative dragon-buff effects (MS, damage amp, mitigation), and order columns by win-correlation at the current game time; partial liveclient rows may show K/D/A, total CS, and vision while marking unsupported damage/gold/objectives unavailable—never synthesize zeros or treat partial coverage as full career.
- Grub-touch estimates must be auditable (assumptions, burn uptime, tick rate, melee/ranged, plate-equivalent) so numbers are reproducible; prefer mining high-confidence proxies (including match VOD analysis) when confidence is low.

## Learned Workspace Facts
- Utility-only abilities (slows, withers, shields, MS buffs) must never be skipped for having zero base damage — e.g. Nasus W / Zilean E reshape AA counts, xH, and kite windows.
- Full game knowledge is ingested via `npm run ingest:lolwiki` into `public/data/lolwiki/` (all Summoner's Rift items only — not TFT/Arena/ARAM, all runes including Unsealed Spellbook, all summoners, all 171 Meraki champion kits). Do not ship a hand-curated subset as the source of truth.
- This repo is a LoL combat strength analysis app (React + TypeScript + Vite) with a phased roadmap: v1 isolated 1v1 calculator → v2 map/game-state import into the calculator → v3 xH/xHm → v4 teamfight win-odds ranges (worst/typical/best).
- xH is skillshot hit chance from similar historic situations (mobility buckets, map zones, only when enemies are in ability range); xHm is multi-target hit distribution; target-side objective-resolved mobility (e.g. Cloud Soul MS) must reach every xH path; current code uses priors until historic rates replace them.
- Timeline ingest uses Riot live-stats JSONL (`events_*_riot.jsonl`, `stats_update` / rfc461Schema) or product `npm run rofl:ingest` into `artifacts/rofl/<matchCode>/`, then published sanitized timelines under `public/data/matches/<matchCode>/` with atomic registry `public/data/matches/index.json` (relative URLs, coverage gates). Canonical rfc461 is the only event interchange. Match identity/display name is the BR1 match code plus ROFL/liveclient roster; join on PUUID/full Riot ID, never participant order. Research fixtures (FUR parity, Maknee) stay demo-labeled and are never product defaults or calculator-trust overrides; never fixture-remap their packet layouts/ranks onto a live match; schema-proof/fixture/invalid entries must not enter the registry. Patch-matched Replay API (`https://127.0.0.1:2999`) yields real positions/levels/items/alive (and liveclient K/D/A, total CS, vision when sampled)—not HP/ranks/combat; unsupported fields stay absent/null, never zero. Replay API capture speed shortcuts (e.g. compact cached selection / zero-settle) stay opt-in experimental and must not become the product default. Native 1Hz positions lerp only across continuous segments and snap annotated discontinuities (no cross-map walks); synthetic waypoint walking is fixture-only; while offline waypoints remain undecoded, product positions stay Replay API–only. FUR live-stats field parity is the bar for ROFL decrypt/packet-drive HP and combat (`docs/rofl-format.md`). Offline ROFL packet/movement/HP research must be clean-room from protocol facts and in-repo probes—do not copy or vendor unlicensed third-party decrypt code or League client binaries. Live BR1 decrypt → `npm run rofl:live-fur` / `rofl:schema-proof` is a fixture-backed schema proof and must not publish as a real match. Real-match calculator readiness (`calculatorReady`) requires same-match identity-bound positions + HP + combat + ranks (`hpKnown`/`combatStatsKnown`/`abilityRanksKnown`) that pass `validate-rofl-pipeline.py --product`; combat wire maps must be PE/string-table proven (CharacterIntermediate object slots are not wire indices). Zaahen is a real champion; Wukong’s internal/asset key is `MonkeyKing` — never rewrite Zaahen to Wukong.
- Product ROFL HP fusion accepts only `rofl-trusted-hp-v1`: exact same-match manifests/hash/roster, ten stable PUUID/full Riot ID → netId bindings, at least two replay-time samples, explicit mMaxHP, and ≤500 ms alignment. Static/CreateHero-order fusion remains research-only; HP never implies combat stats or ability ranks.
- Ability ranks from live same-match packet decode are required for correct scalings (wiki kit data alone is not enough); set `abilityRanksKnown` only when proven, never by remapping fixture ranks.
- Void grub and dragon/Baron/Elder effects must follow wiki-accurate rules and feed combat only when honestly modelable; a team's fourth elemental dragon is a real permanent stack that also grants Soul. Timed/conditional soul and Elder effects stay disclosed-not-faked. Authoritative live timeline AD/AP/armor/MR/AS/MS already include objective buffs—do not re-apply Infernal/Mountain/Hex/Baron/Cloud Soul on those pins (`docs/objective-model.md`).
- Jungle camp spawn/respawn overlays must use current wiki timings (early-game camp clocks have changed); availability on the map must be second-precise from the timeline.
- VOD vision work lives under `vision/` with ~0.5s ffmpeg frame sampling and bbox object-detection labeling; annotation remains manual for now with Label Studio export structure kept for later revisit.
- CORE/hand-curated champion kits mean modeling attention only—not patch-validated trust; `modelTrust` distinguishes manual-kit 1v1 vs experimental NvM/generated, with calibrated=false until empirical validation (`docs/combat-trust-boundary.md`, coverage audit via `scripts/audit-combat-coverage.ts`).
- Timed cast/AA rotation planning applies to living 1v1 with a resolvable kit (CORE or Meraki/generated): attack-reset empowered autos, stop at first lethal (death-coupled; `estimateSurviveSec` is aggregate/NvM-only), auditable omit/stop notes, xH-aware target-effective scoring, and front-loaded planner scoring so longer windows do not AA-pad and park abilities after first-lethal truncate; short-window engage stays timed (opener at t=0 on the clock; defender reaction delay + no engageCc); Meraki stays experimental/uncalibrated with default execution timing disclosed; NvM and unresolved kits keep the aggregate-window fallback.
